# 第三篇：AI Agent 的心脏

## ——一个永不停歇的对话循环

---

上一篇我们看了 Claude Code 的整体架构。这一篇，我们深入最核心的部分：**对话循环**。

当你在终端输入"帮我重构这个函数"，Claude Code 是怎么一步步处理的？

---

## 从一次对话说起

假设你输入：**"帮我找出 src/ 目录下所有超过 200 行的文件"**

Claude Code 的处理过程大概是这样的：

```
你的输入
  ↓
发给 AI 模型
  ↓
模型说："我需要用 GlobTool 搜索文件"
  ↓
执行 GlobTool
  ↓
把结果发回给模型
  ↓
模型说："我需要用 BashTool 统计行数"
  ↓
执行 BashTool
  ↓
把结果发回给模型
  ↓
模型给出最终答案
  ↓
显示给你
```

这个"发给模型 → 模型调用工具 → 执行工具 → 发回结果 → 再发给模型"的循环，就是 AI Agent 的核心。

---

## 核心代码：query.ts

这个循环的实现在 `src/query.ts`，大约 1700 行代码。简化后的骨架是这样的：

```typescript
// 核心循环（简化版）
async function* queryLoop(params) {
  let messages = params.messages  // 对话历史

  while (true) {  // 永远循环，直到没有工具调用
    // 第一步：调用 AI 模型，获取流式响应
    const toolUseBlocks = []
    for await (const event of callModel(messages)) {
      yield event  // 把每个 token 实时传给 UI 显示

      if (event.type === 'tool_use') {
        toolUseBlocks.push(event)  // 记录工具调用
      }
    }

    // 第二步：如果没有工具调用，结束循环
    if (toolUseBlocks.length === 0) {
      break
    }

    // 第三步：执行所有工具
    const toolResults = await runTools(toolUseBlocks)
    yield toolResults  // 把工具结果传给 UI 显示

    // 第四步：把工具结果加入对话历史，继续循环
    messages = [...messages, ...toolResults]
  }
}
```

就这么简单。一个 while 循环，不断地"调用模型 → 执行工具 → 把结果加入历史 → 再调用模型"。

---

## 流式输出是怎么工作的？

注意上面代码里的 `yield event`。

AI 模型不是一次性返回完整答案的，而是一个字一个字地"流"出来。就像你看到 ChatGPT 的回答是一个字一个字出现的，Claude Code 也是这样。

```typescript
// 模型返回的是一个流，每次 yield 一个 token
for await (const event of callModel(messages)) {
  if (event.type === 'content_block_delta') {
    // 这是一个 token（可能是一个字、一个词、一个标点）
    yield event  // 立刻传给 UI，UI 立刻显示
  }
}
```

这就是为什么你能看到 Claude Code 的回答是实时"打字"出来的，而不是等待一段时间后突然全部出现。

## 工具调用的完整流程

让我们更详细地看工具调用是怎么工作的。

**第一步：模型决定调用工具**

模型的回复不只是文字，还可以包含"工具调用"：

```json
{
  "type": "tool_use",
  "name": "Bash",
  "input": {
    "command": "find src/ -name '*.ts' | xargs wc -l | sort -n"
  }
}
```

**第二步：Claude Code 执行工具**

```typescript
// src/services/tools/toolOrchestration.ts（简化）
async function runTools(toolUseBlocks) {
  const results = []

  for (const toolUse of toolUseBlocks) {
    // 先检查权限
    const permission = await canUseTool(toolUse)
    if (permission.behavior === 'deny') {
      results.push({ error: '权限被拒绝' })
      continue
    }

    // 执行工具
    const tool = findToolByName(toolUse.name)
    const result = await tool.call(toolUse.input)
    results.push(result)
  }

  return results
}
```

**第三步：把结果发回给模型**

```json
{
  "role": "user",
  "content": [
    {
      "type": "tool_result",
      "tool_use_id": "xxx",
      "content": "找到 15 个文件，其中 3 个超过 200 行：\n- src/query.ts (1733行)\n- ..."
    }
  ]
}
```

**第四步：模型根据结果继续回答**

模型看到工具结果后，可能继续调用其他工具，也可能直接给出最终答案。

---

## 一个容易忽视的细节：并行工具调用

Claude Code 支持**同时调用多个工具**。

比如模型可能说："我需要同时读取 3 个文件"，然后一次性发出 3 个 `FileRead` 工具调用。Claude Code 会并行执行这 3 个工具，而不是一个一个等待：

```typescript
// 并行执行所有工具调用
const results = await Promise.all(
  toolUseBlocks.map(toolUse => executeTool(toolUse))
)
```

这大大提高了效率。如果串行执行，3 个文件读取可能需要 300ms；并行执行只需要 100ms。

---

## 流式降级：当流式响应失败时

有时候网络不稳定，流式响应会中断。Claude Code 有一个优雅的降级机制：

```typescript
if (streamingFallbackOccurred) {
  // 清理已经显示的不完整内容
  for (const msg of assistantMessages) {
    yield { type: 'tombstone', message: msg }  // 告诉 UI 删除这些消息
  }

  // 重新用非流式模式请求
  // 非流式模式：等待完整响应后一次性返回
  const response = await callModelNonStreaming(messages)
  yield response
}
```

`tombstone`（墓碑）是一个很形象的命名——把之前显示的不完整内容"埋掉"，重新来过。

---

## 最大轮次限制

为了防止 Agent 无限循环，有最大轮次限制：

```typescript
// 默认最多 10 轮工具调用
const maxTurns = params.maxTurns ?? 10

if (turnCount >= maxTurns) {
  yield { type: 'attachment', attachment: { type: 'max_turns_reached' } }
  break
}
```

如果 Agent 调用了 10 次工具还没完成任务，就会停下来告诉你"已达到最大轮次"。

## 整个流程的时序图

```
用户输入
  │
  ▼
[query.ts] 开始循环
  │
  ├─→ [claude.ts] 调用 API（流式）
  │     │
  │     ├─→ yield token1 → UI 显示
  │     ├─→ yield token2 → UI 显示
  │     ├─→ yield tool_use → 记录工具调用
  │     └─→ yield message_stop → 流结束
  │
  ├─→ [toolOrchestration.ts] 执行工具
  │     │
  │     ├─→ 检查权限
  │     ├─→ 并行执行工具
  │     └─→ 返回结果
  │
  ├─→ 把工具结果加入消息历史
  │
  └─→ 继续循环（或结束）
```

---

## 小结

Claude Code 的核心循环其实很简单：

1. **调用模型**，获取流式响应
2. **实时显示** token
3. **收集工具调用**
4. **执行工具**（并行）
5. **把结果加入历史**
6. **回到第 1 步**，直到没有工具调用

这个模式不是 Claude Code 独有的，几乎所有 AI Agent 框架（LangChain、AutoGen、CrewAI）的核心都是这个循环。理解了这个，你就理解了 AI Agent 的本质。
