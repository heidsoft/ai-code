# 第二篇：Claude Code 到底是什么？

## ——一个跑在终端里的 React 应用

---

上一篇讲了 Claude Code 源码泄漏的来龙去脉。这一篇，我们来看看这份代码到底揭示了什么。

很多人以为 Claude Code 只是一个简单的命令行工具，把你的问题发给 API，再把回答打印出来。

**实际上完全不是这样。**

---

## 第一个惊喜：它是用 React 写的

打开源码，第一个让人意外的发现是：**Claude Code 的终端界面是用 React 写的**。

不是网页版的 React，而是通过一个叫 [Ink](https://github.com/vadimdemedes/ink) 的框架，把 React 渲染到终端里。

```tsx
// src/screens/REPL.tsx（简化）
function REPLScreen() {
  const [messages, setMessages] = useState([])
  const [input, setInput] = useState('')

  return (
    <Box flexDirection="column">
      <Messages messages={messages} />
      <PromptInput value={input} onChange={setInput} />
    </Box>
  )
}
```

这意味着：
- 终端里的每一行输出，都是一个 React 组件
- 有完整的状态管理（useState、useEffect）
- 甚至用了 **React Compiler** 做性能优化

为什么要这么做？因为 Claude Code 的界面比想象中复杂：工具调用的进度条、权限确认对话框、多 Agent 的状态显示……用 React 管理这些状态比手写字符串拼接要清晰得多。

---

## 第二个惊喜：运行时是 Bun，不是 Node.js

```json
// package.json
{
  "scripts": {
    "dev": "bun run src/entrypoints/cli.tsx",
    "build": "bun build src/entrypoints/cli.tsx --outdir dist --target bun"
  }
}
```

Anthropic 选择了 [Bun](https://bun.sh) 而不是 Node.js。

原因很直接：**启动速度**。Claude Code 是一个交互式工具，每次打开都要等待启动，Bun 的启动速度比 Node.js 快 3-4 倍，用户体验差别明显。

最终打包成一个约 25MB 的单文件，这就是为什么 `npm install -g @anthropic-ai/claude-code` 之后可以直接用 `claude` 命令。

---

## 整体架构：六层结构

从源码来看，Claude Code 分为六个清晰的层次：

```
┌─────────────────────────────────────┐
│  UI 层（终端界面）                    │
│  ink/ + components/ + screens/      │
├─────────────────────────────────────┤
│  命令层（CLI 子命令）                 │
│  commands/（80+ 个命令）             │
├─────────────────────────────────────┤
│  Agent 编排层                        │
│  coordinator/ + tasks/              │
├─────────────────────────────────────┤
│  工具层（AI 可调用的能力）            │
│  tools/（40+ 个工具）                │
├─────────────────────────────────────┤
│  服务层（API、压缩、OAuth 等）        │
│  services/                          │
├─────────────────────────────────────┤
│  基础设施层（状态、会话、桥接）       │
│  bridge/ + state/ + bootstrap/      │
└─────────────────────────────────────┘
```

## 启动流程：极致的性能优化

`src/entrypoints/cli.tsx` 是真正的入口文件。里面有一个很有意思的设计——**分级快速路径**：

```typescript
async function main() {
  const args = process.argv.slice(2)

  // 快速路径：--version 完全不加载任何模块
  if (args[0] === '--version') {
    console.log(`${MACRO.VERSION} (Claude Code)`)
    return
  }

  // 快速路径：--help 只加载命令定义
  // 快速路径：bridge 模式只加载桥接模块
  // ...

  // 最后才加载完整 CLI（几十个模块）
  const { main: cliMain } = await import('../main.jsx')
  await cliMain()
}
```

`--version` 命令完全不加载任何模块，直接输出版本号返回。这样即使是最简单的命令也能在几毫秒内响应。

## Feature Flag 系统：大量功能被隐藏

源码里有一个关键的 polyfill：

```typescript
// src/entrypoints/cli.tsx 顶部
const feature = (_name: string) => false
```

原始代码里大量使用 `feature('FLAG_NAME')` 来控制功能开关：

```typescript
// 这些功能在泄漏版本里全部被关闭
if (feature('KAIROS')) { /* 后台 Agent 模式 */ }
if (feature('COORDINATOR_MODE')) { /* 多 Agent 协调 */ }
if (feature('PROACTIVE')) { /* 主动建议 */ }
if (feature('DREAM_MODE')) { /* 持续后台推理 */ }
```

这是 Anthropic 内部的 Feature Flag 系统，通过 GrowthBook（一个 A/B 测试平台）控制。在泄漏的版本里，所有 flag 都返回 `false`，意味着这些功能的代码虽然存在，但全部是死代码。

这也解释了为什么 Claude Code 的功能比代码里看到的要少——很多功能还在内部测试阶段。

## 配置系统：自动读取 `~/.claude/settings.json`

这是很多用户不知道的细节。Claude Code 启动时会自动读取 `~/.claude/settings.json`，并把 `env` 字段注入到环境变量：

```typescript
// src/utils/envUtils.ts
export const getClaudeConfigHomeDir = memoize(() => {
  return process.env.CLAUDE_CONFIG_DIR ?? join(homedir(), '.claude')
})
```

所以只需要在配置文件里写：

```json
{
  "env": {
    "ANTHROPIC_BASE_URL": "https://api.minimaxi.com/anthropic",
    "ANTHROPIC_AUTH_TOKEN": "your-token",
    "ANTHROPIC_MODEL": "MiniMax-M2.7"
  }
}
```

就能让 Claude Code 使用任何兼容 Anthropic API 格式的模型（比如 MiniMax、DeepSeek），完全不需要修改代码。

---

## 支持多个云服务商

`src/services/api/claude.ts` 是 API 客户端，支持四个提供商：

- **Anthropic 直连**（默认）
- **AWS Bedrock**
- **Google Vertex AI**
- **Azure**

通过 `ANTHROPIC_BASE_URL` 环境变量，可以把请求转发到任何兼容 Anthropic API 格式的服务。这就是为什么国内开发者可以用 MiniMax 等模型来驱动 Claude Code。

## 这份代码能跑起来吗？

可以。泄漏的代码虽然有约 1341 个 TypeScript 类型错误（反编译导致的类型信息丢失），但不影响运行时执行。

```bash
# 安装依赖
bun install

# 测试
echo "你好" | bun run src/entrypoints/cli.tsx -p
# 输出：你好！有什么我可以帮助你的吗？

# 查看版本
bun run src/entrypoints/cli.tsx --version
# 输出：2.1.888 (Claude Code)
```
