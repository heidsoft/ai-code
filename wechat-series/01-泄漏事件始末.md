# 第一篇：Anthropic 一次失误，51万行代码曝光

## ——Claude Code 源码泄漏始末

---

2026 年 3 月 31 日早上，一条推文在 AI 圈炸开了锅。

安全研究员 Chaofan Shou 发帖：

> "Claude Code source code has been leaked via a map file in their npm registry!"

这条帖子最终获得了超过 **2800 万次浏览**。

---

## 事情是怎么发生的？

Claude Code 是 Anthropic 开发的 AI 编程助手，运行在终端里，可以帮你读代码、改代码、执行命令。它是闭源的，代码经过混淆处理，正常情况下没人能看到内部实现。

但 2026 年 3 月 30 日，Anthropic 发布了 v2.1.88 版本。这次发布犯了一个低级错误：**把一个 57MB 的 source map 文件（`cli.js.map`）一起打包进了 npm 包**。

### 什么是 source map？

开发者写的代码经过编译、压缩之后，变成了机器友好但人类难以阅读的混淆代码。source map 就像一张"还原地图"，记录了混淆代码和原始代码之间的对应关系。

正常情况下，source map 只在开发调试时使用，**绝对不应该出现在发布的生产包里**。

但这次，它出现了。

### 任何人都能拿到

不需要黑客技术，不需要特殊权限。只需要：

```bash
npm install @anthropic-ai/claude-code
```

然后在 `node_modules` 里就能找到这个文件，里面包含了完整的、未混淆的 TypeScript 源代码。

---

## 泄漏了什么？

这个 source map 文件映射了：
- **约 2000 个 TypeScript 文件**
- **超过 51.2 万行代码**

包含的内容：
- 完整的 AI 对话循环实现
- 所有工具（文件读写、命令执行、网络搜索等）的实现细节
- 权限控制系统
- 多 Agent 协作机制
- **几个从未对外公布的内部功能**（后面会讲）

---

## 社区的反应

Chaofan Shou 发帖后，GitHub 上迅速出现了多个镜像仓库。其中一个在短时间内积累了超过 **8.4 万 star 和 8.2 万 fork**，成为当时增长最快的 GitHub 仓库之一。

Anthropic 当天就下架了这个版本，并发表声明：

> "没有敏感客户数据或凭证被暴露。这是一次由人为失误导致的发布打包问题，不是安全漏洞。"

但为时已晚。代码已经被镜像，永久留在了互联网上。

值得一提的是，这已经是 Claude Code **一年内第二次**通过 source map 泄漏源码。

---

## 泄漏的安全影响

这次泄漏不只是知识产权问题。

**1. 攻击面暴露**

AI 安全公司 Straiker 指出，攻击者现在可以研究数据在 Claude Code 内部的流转方式，精心构造能在上下文压缩后依然存活的恶意 payload，实现跨会话持久化攻击。

**2. 供应链攻击**

泄漏暴露了一些从未公开的内部依赖包名。这些包名在 npm 上没有被注册，于是攻击者在泄漏发生后数小时内就注册了同名的恶意包，等待开发者误装。

**3. 恶意软件利用**

安全公司 Zscaler 发现，有人在 GitHub 上创建了伪装成"Claude Code 泄漏源码"的仓库，实际上是传播 Vidar 信息窃取木马和 GhostSocks 代理恶意软件。这些仓库甚至出现在 Google 搜索结果的前列。

---

## 几个从未公开的内部功能

这次泄漏最让社区兴奋的，是发现了几个 Anthropic 从未对外宣布的功能：

**KAIROS — 持久后台 Agent**

一个可以在后台自主运行任务的模式，能在不等待用户输入的情况下自动修复错误，并通过推送通知告知用户。

**Undercover Mode — 卧底模式**

代码里有这样一段系统提示：

> "You are operating UNDERCOVER in a PUBLIC/OPEN-SOURCE repository. Your commit messages, PR titles, and PR bodies MUST NOT contain ANY Anthropic-internal information. Do not blow your cover."
>
> （你正在一个公开/开源仓库中**卧底**运行。你的 commit 信息、PR 标题和 PR 描述中绝对不能包含任何 Anthropic 内部信息。不要暴露你的身份。）

**Anti-distillation 投毒**

代码中存在向 API 响应注入虚假工具定义的机制，用于干扰竞争对手抓取 Claude Code 输出来训练自己模型的行为。

## 对开发者的警示

这次事件的根本原因很简单：**发布 npm 包时没有检查 source map 是否被包含进去**。

防止这类问题的方法：

```bash
# 发布前检查，发现 .map 文件就中止
if find dist/ -name "*.map" | grep -q .; then
  echo "ERROR: sourcemap files found. Aborting publish."
  exit 1
fi
```

或者在 `package.json` 里用白名单明确控制发布内容：

```json
{
  "files": [
    "dist/**/*.js",
    "dist/**/*.d.ts"
  ]
}
```

一个遗漏的配置项，51 万行代码的代价。
